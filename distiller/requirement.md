# 要件定義書 — Distiller

## 1. 概要

Distiller は、指定された Web サイトの URL から MHTML を生成し、コンテンツの種別判定と主要コンテンツの抽出を行うシステムである。  
同一サイトへの再リクエスト時には過去の抽出結果を活用し、効率的に再抽出を行う。

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| 原液（サイト） | 抽出対象となる Web ページの URL |
| ディスティール | MHTML の取得・種別判定・コンテンツ抽出の一連の処理 |
| ディスティール結果 | 過去に実行した抽出処理の成果物および付随するメタ情報 |

## 3. 機能要件

### 3.1 MHTML 生成

- 入力として URL を受け取る。
- 指定された URL の Web ページを取得し、MHTML 形式で保存する。

### 3.2 コンテンツ種別判定

MHTML の内容を分析し、以下のいずれかの種別を判定する。

| 種別 | 説明 |
|---|---|
| レシピ | 料理・調理手順を含むコンテンツ |
| ニュース | 報道・時事情報を含むコンテンツ |
| その他 | 上記に該当しないコンテンツ |

### 3.3 主要コンテンツ抽出・HTML 生成

- 判定した種別に応じた抽出ロジックを適用し、MHTML から主要コンテンツを抽出する。
- 抽出したコンテンツを含む新たな HTML を生成する。
- 種別ごとに抽出対象となる要素が異なる（例: レシピなら材料・手順、ニュースなら本文・見出し）。

### 3.4 再リクエスト時の差分抽出

- 同一 URL（原液）に対して再度リクエストを受信した場合、以下の動作を行う。
  1. 過去のディスティール結果（種別判定・抽出コンテンツ・メタ情報）を参照する。
  2. 過去の結果を踏まえ、必要な情報を改めて取得し再抽出を行う。
  3. 新たなディスティール結果として保存・返却する。

## 4. 処理フロー

```
URL 受信
  │
  ├─ 同一 URL の過去結果が存在するか？
  │    ├─ YES → 過去のディスティール結果を参照し再抽出
  │    └─ NO  → 新規処理
  │
  ▼
MHTML 生成
  │
  ▼
コンテンツ種別判定（レシピ / ニュース / その他）
  │
  ▼
種別に応じた主要コンテンツ抽出
  │
  ▼
抽出コンテンツを含む HTML 生成
  │
  ▼
ディスティール結果の保存・返却
```

## 5. 非機能要件

| 項目 | 内容 |
|---|---|
| 保存 | ディスティール結果は永続化し、再リクエスト時に参照可能とする |
| 拡張性 | 種別（レシピ・ニュース・その他）は将来的に追加可能な設計とする |
| 冪等性 | 同一 URL・同一コンテンツに対する処理結果は一貫性を持つ |

## 6. 未決事項

- MHTML 生成に使用するブラウザエンジン／ライブラリの選定
- 種別判定のアルゴリズム（ルールベース／機械学習）の決定
- 再リクエスト時に「再抽出が必要」と判断する基準の定義
- ディスティール結果の保存先（ファイルシステム／DB）の決定
- 生成する HTML のテンプレート仕様の策定
